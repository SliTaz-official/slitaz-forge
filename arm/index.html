SliTaz ARM

Welcome to SliTaz ARM

This website provide informations, notes and links about the SliTaz GNU/Linux port to the ARM architecture.

Packages

All SliTaz ARM packages are automaticaly built by the ARM Cooker. To properly handle SLITAZ_ARCH variable the recommended Packages Manager is spk. One way to start with the Spk Toolset is to generate a distro and include latest Hg version with 'sat gen --spk' and then emulate the system to try it out.

Run 'spk-up --list' to get the latest packages list.

Current ARM packages mirror URL: http://cook.slitaz.org/cross/arm/packages/

SliTaz ARM Tools

To create some custom ARM SliTaz flavors we use a tool called 'sat' witch stand for SliTaz ARM Tool. Sat is a cmdline tool all written in SHell script and in the spirit of SliTaz, light, fast, easy to use! You may want to browse slitaz-arm Hg repo and have a look to the README. 

$ hg clone http://hg.slitaz.org/slitaz-arm

Sat in under heavy development so dont forget to update your Hg repo - March 2014

Raspberry Pi

Currently SliTaz official ARM port works on the Raspberry Pi and can be used to setup a small server. DirectFB is also ported and provide nice and light GUI support. A new tool is on the stove to create custom Rpi flavors with official Kernel, cutom boot options and more.

Official Raspberry Pi website: www.raspberrypi.org

Prebuilt ARM Toolchain

To compile SliTaz packages to ARM you need a working cross compilation toolchain. You can built your own with 'cross' or use our prebuilt toolchains avalaible from SliTaz Mirror: http://mirror.slitaz.org/packages/cross/

Cross compiling

SliTaz provide all the necesary tools for cross compiling Open Source software. The cookutils package provide 'cross' witch is used to create cross compilation toolchains and 'cook' is used to cross compile SliTaz packages.

README, howto and doc: cross - cook
